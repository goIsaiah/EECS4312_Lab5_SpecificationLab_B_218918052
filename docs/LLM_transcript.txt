Prompt 2:
I did pip install pytest, and just pytest in the terminal and it's still not working

LLM Response 2:
This is a very common hiccup when using Python on Windows! Don't worry, you didn't do anything wrong.

When you get an error after typing `pytest` (usually saying it's "not recognized as an internal or external command"), it almost always means Windows doesn't know *where* to look for the `pytest` program, even though `pip` successfully installed it.

Here is the quickest way to bypass this issue, along with a couple of things to check.

### The Quickest Fix: Use the Python Module Command

Instead of asking Windows to find `pytest` directly, you can ask Python to run it for you. Since Windows already knows where Python is, this works 99% of the time.

In your VS Code terminal, run this exact command instead:

```powershell
python -m pytest

```

*(Or, for the detailed output, use: `python -m pytest -v`)*

**Why this works:** The `-m` flag tells Python, "Hey, look through the modules you have installed, find the one called `pytest`, and run it."

---

### Why didn't `pytest` work on its own?

There are usually two culprits for this on Windows and VS Code:

**1. The Windows PATH Variable**
When `pip` installs programs like `pytest`, it puts them in a special "Scripts" folder. If that folder isn't added to your Windows "Environment Variables" (specifically the `PATH`), your terminal simply can't "see" the command. Using `python -m pytest` safely ignores this problem.

**2. Virtual Environment Mismatch**
VS Code might be trying to be helpful by automatically using a "Virtual Environment" (a segmented, clean installation of Python just for this project), but you might have installed `pytest` to your global, main Python installation instead.

**How to check your environment in VS Code:**

1. Look at the **bottom-right corner** of your VS Code window.
2. You should see a version number for Python (e.g., `3.12.1`). Click on it.
3. A menu will pop up at the top of the screen asking you to **Select Interpreter**.
4. Make sure you select the one labeled "Recommended" or the one where you remember installing your tools. If the terminal you have open doesn't match this environment, you might need to close the terminal (click the trash can icon) and open a new one (`Ctrl` + ```) so VS Code can load the correct settings.
